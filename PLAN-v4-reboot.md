## 1. Scope

A coordinated upgrade of every layer in the NemoClaw-Thor stack, followed by
validation of the full agentic workflow end-to-end. This is not incremental
patching — it is a controlled reboot that brings every component to current,
adapts scripts to upstream API changes, and re-validates the agent pipeline.

### Phase E — NemoClaw-Thor Script Refactoring ✓ COMPLETE (2026-04-06)

Scripts were stripped to proven-necessary only. Deleted files are recoverable
from git history.

#### E.1: Legacy launcher cleanup ✓

Deleted: `start-qwen3.5-*.sh` (4 thin wrappers), `nemotron3-thor*.sh` (2
obsolete Nemotron launchers). `start-model.sh <profile>` is the only launcher.

#### E.2: Proxy removal ✓

Deleted: `lib/restream-proxy.py`, `lib/nonstream-proxy.py`. All proxy code
removed from `sandbox-runtime.sh`. Inference now routes through OpenShell
provider (`https://inference.local/v1`) — no proxy in the data path.

**Key v3→v4 difference**: v3 used a restream proxy on port 8199 inside the
sandbox pod (root namespace) to buffer tool-call fragments. OpenClaw talked
to the proxy, which forwarded to host vLLM. This caused namespace isolation
issues (SSH namespace couldn't reach pod root namespace). v4 eliminates the
proxy entirely — OpenClaw talks to `inference.local` via the OpenShell
provider route, which handles the network path natively.

#### E.3: Script pruning ✓

Deleted 17 scripts not exercised in the v4 session: `install.sh`,
`uninstall.sh`, `check-prerequisites.sh`, `apply-host-fixes.sh`,
`backup-host-state.sh`, `restore-host-state.sh`, `start-gateway.sh`,
`start-dashboard-access.sh`, `stop-dashboard-access.sh`,
`apply-policy-profile.sh`, `apply-policy-additions.sh`,
`reset-sandbox-session-state.sh`, `install-node.sh`, `uninstall-node.sh`,
`lib/host-state.sh`, `lib/test-checks.sh`, `lib/policy.sh`.

Remaining scripts (4 + 5 libs):
- `start-model.sh` — launch vLLM
- `configure-local-provider.sh` — wire OpenShell provider + patch openclaw.json
- `status.sh` — health checks
- `enforce-egress-firewall.sh` — standalone iptables control
- `lib/checks.sh`, `lib/config.sh`, `lib/launch.sh`, `lib/sandbox-runtime.sh`, `lib/egress-firewall.sh`

#### E.4: configure-local-provider.sh — PARTIAL

## 3. Risk Register
